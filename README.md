## 🌌 More about me &nbsp;<sup>_click to expand_</sup>

<details>
  <summary><b>⚡ &nbsp;What I'm learning right now</b></summary>
  <br/>

  - 🔴 **Offensive** — web exploitation, binary exploitation, recon & tooling
  - 🔵 **Defensive** — detection engineering, hardening, WAF & firewall internals
  - 🧪 **Building** — turning what I learn into security tools that actually ship
  - 🎯 **Sharpening** — bug bounty on Intigriti, boxes on HackTheBox, and CTFs
</details>

<details>
  <summary><b>🧠 &nbsp;How I approach security</b></summary>
  <br/>

  > **"Learn how it breaks. Build what defends it."**

  I don't pick a lane — offense and defense feed each other. Every exploit I study becomes a detection I can write;
  every defense I build gets stress-tested from the attacker's chair. And whenever a problem is annoying enough to
  keep hitting, I'd rather **write the tool** than keep working around it.
</details>
